# 后端项目结构

1. `db.py`: 全局数据库，每个子系统都需要引入

2. \`_\_init\_\_.py`:  工厂方法运行：

   <font color=#ff00>在backend的上级目录运行！</font>

   ```shell
   $ set FLASK_APP=backend
   $ flask run
   ```

3. `config.py`：后端配置文件

4. `templates`: 静态资源（无用）

5. `static`: 静态资源（无用）

6. `dish`: 

   * `__init__.py`:

     定义注册蓝图函数，在`../__init__.py`中统一注册

   * ``dish.py`

     由路由实现前后端数据库交互

   * `models.py`

     数据库表的ORM定义，我抽空完成，不用管。
